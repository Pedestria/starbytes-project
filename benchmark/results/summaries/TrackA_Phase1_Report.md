# Track A Phase 1 Report

Status: Implemented.

## Scope Completed

- Shared workload contract and locked input data are defined in:
  - `benchmark/workloads/track_a/contracts.json`
  - `benchmark/data/track_a/track_a_inputs.json`
  - `benchmark/data/track_a/dna_input.fasta`
- Track A workloads are implemented in all four language sets:
  - `binary-trees`
  - `n-body`
  - `spectral-norm`
  - `fasta`
  - `k-nucleotide`
  - `regex-redux`
- Reproducible runner entrypoints now exist:
  - `benchmark/runners/run_track_a.sh`
  - `benchmark/runners/run_all.sh`
  - `benchmark/runners/run_track_a.py`
- Standardized summary/report output is generated by:
  - `benchmark/runners/summarize_track_a.py`
  - `benchmark/results/summaries/track_a_summary.schema.json`

## Run Commands

```bash
# TTFR mode
./benchmark/runners/run_track_a.sh --mode ttfr --runs 20 --warmup 3

# Steady-state mode
./benchmark/runners/run_track_a.sh --mode steady-state --runs 20 --warmup 3

# All currently implemented Track A runs
./benchmark/runners/run_all.sh --runs 20 --warmup 3
```

Smoke validation when `hyperfine` is unavailable:

```bash
./benchmark/runners/run_track_a.sh --mode ttfr --smoke
```

## Generated Artifacts

- raw hyperfine JSON: `benchmark/results/raw/track_a_<mode>_<timestamp>/`
- summary JSON: `benchmark/results/summaries/track_a_<mode>_<timestamp>.summary.json`
- report markdown: `benchmark/results/summaries/track_a_<mode>_<timestamp>.report.md`

## Phase 1 Notes

- Track A uses shared single-threaded workload sizes that are practical for the current Starbytes runtime while remaining identical across languages.
- Starbytes `steady-state` currently uses a warmed compile/run fallback because the toolchain does not yet expose a standalone compiled-module executor.
- Full measured runs still depend on host availability of `hyperfine`, `go`, `rustc`, and the built Starbytes toolchain.
